Checkpoints:  Test Script

    • Sufficient test scripts been implemented to achieve acceptable test coverage. See Section: Key Measures of Testing.
    • Each non-shell test script accurately implements only one (or part of one) test procedure.
    • The test scripts accurately reflect the test model.
    • Each test script has been played back / debugged to ensure proper execution.
    • Each test script containing a verification method, or branching logic has been played back / debugged under both positive and negative conditions to ensure the test scripts and verification methods execute as expected and desired.
    • The test scripts have been recorded / programmed in the appropriate manner (to increase test script re-use and minimize test script maintenance).
    • All the test script names and / or IDs are consistent with your test artifact naming convention.
